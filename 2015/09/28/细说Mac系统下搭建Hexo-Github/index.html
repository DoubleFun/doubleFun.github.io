细说Mac系统下搭建博客Hexo+Github

  • Hexo

前言

最近突然有个想法：想搭建自己的Blog，记录在工作或学习过程中的一些点滴；于是乎各种百度 or Google,最终决定使用Hexo+Github的方式来搭建和部署自己的博客网站。(注意：此教程基于Mac系统, Linux/Windows 用户仅供部分参考,可以自己去百度搜索!)

###二.开始安装: step by step (以下相关的命令都是在mac命令行Terminal下执行)

####1.Git安装
brew install git #比较简单 一行命令搞定

####2.安装Node.js (相关介绍：https://github.com/creationix/nvm#node-version-manager 我在安装过程中使用了vpn还是蛮快的，安装过程中如果出错,请参考以上链接.)
brew install node # 最新版的node.js的包中已经集成了npm包管理工具
使用以下命令验证是否安装成功
node -v
npm -v
比如我的终端上显示:
JerryS-Air:_posts just$ node -v
v0.12.7
JerryS-Air:_posts just$ npm -v
2.14.2

####3.Hexo安装与设置

#####在介绍安装之前，我们先简单了解下hexo;

######1.关于Hexo 中文版

######2. hexo系列教程(一)～(五)

######3. 简单了解后,我们开始继续安装: Node, npm和Git都安装成功后, 开始安装hexo
npm install hexo -g #-g表示全局安装

到此基本上安装完毕，下面开始设置相关的配置。配置前首先需要在本地创建你的博客对应的文件夹，比如我自己是在桌面创建的DoubleFun_HexoBlog.然后通过下面的几个命令完成初始化操作.


cd DoubleFun_HexoBlog/ # 进入到创建的目录文件夹
ls DoubleFun_HexoBlog/ # 查看目录下的内容;此时由于还没有初始化,目录为空
hexo init # 初始化目录
npm install # 初始化依赖

然后查看生成的目录结构如下:

├── _config.yml
├── package.json
├── scaffolds
├── scripts
├── source
|   ├── _drafts
|   └── _posts
└── themes

生成静态文件以及启动本地服务


hexo generate # 生成静态文件
hexo server # 启动本地服务器

然后在你的浏览器中输入http://localhost:4000或者http://0.0.0.0:4000/ 就可以访问刚刚搭建的最原始的博客了.

4. 部署hexo到github

#####以上内容都是在本地进行查看, 现在将博客部署到github上,有以下几个小步骤

(1) 在自己的github账号中创建Repository（没有账号请赶紧创建去吧～）;

Repository name需要符合username.github.io的命名规则。比如我的Github账号是DoubleFun，我就创建名为depthlove.github.io 的仓库;然后访问我仓库的全名称地址是:https://github.com/DoubleFun/doubleFun.github.io

(2) 修改_config.yml文件

在终端上执行命令


vim _config.yml

找到


deploy:
type:

修改为：


deploy:
type: git
repo: https://github.com/DoubleFun/doubleFun.github.io.git
branch: master
保存退出

(3)设置SSH keys

查看是否已经有SSH keys，如果之前在该电脑上使用过git管理过代码，结果是~/.ssh中会存在 id_rsa id_rsa.pub known_hosts 三个文件
在终端执行命令:

ls -al ~/.ssh，
total 48
drwx------   6 just  staff   204  9 25 14:24 .
drwxr-xr-x+ 36 just  staff  1224  9 28 21:49 ..
-rw-r--r--@  1 just  staff  6148  9 25 14:24 .DS_Store
-rw-------   1 just  staff  1675  9 25 14:15 id_rsa
-rw-r--r--@  1 just  staff   406  9 25 14:18 id_rsa.pub
-rw-r--r--   1 just  staff   407  9 25 14:24 known_hosts

如果没有则终端继续执行命令 (dingfan602589919@163.com 是注册github账号时所用的邮箱)

ssh-keygen -t rsa -C "dingfan602589919@163.com"

执行上面的命令后，会出现一些提示，什么都不用管，直接回车，回车，再回车；然后继续输入命令

ssh-agent -s

结果为

SSH_AUTH_SOCK=/var/folders/0r/y1__8yjx579743zfttfrflsr0000gq/T//ssh-a9zU09oKAnpw/agent.17852;
export SSH_AUTH_SOCK;
SSH_AGENT_PID=17853; export SSH_AGENT_PID;
echo Agent pid 17853;

继续输入命令

ssh-add ~/.ssh/id_rsa

结果为

Identity added: /Users/just/.ssh/id_rsa (/Users/just/.ssh/id_rsa)

如果执行ssh-add ~/.ssh/id_rsa 出错，就输入以下指令：

eval `ssh-agent -s`
ssh-add

现在就成功生成了 SSH keys。把SSH keys添加到github账户中。

前往文件夹 ~/.ssh，用文本编辑器打开文件id_rsa.pub，复制其中的内容，然后粘贴到github add SSH keys的文本框中。操作完之后，github会要求输入github的登陆密码，输入完密码就成功添加了 SSH keys。
若还不放心，还可以通过继续在终端执行命令

ssh -T git@github.com  #命令查看SSH keys是否添加成功，

结果如下

Are you sure you want to continue connecting (yes/no)?yes // 输入yes
Warning: Permanently added the RSA host key for IP address '192.xx.xxx.xxx’ to the list of known hosts.
Hi doubleFun! You've successfully authenticated, but GitHub does not provide shell access.

到这里准备工作都已做好，现在就是部署了。继续在终端执行命令

hexo generate
hexo deploy

结果出现错误

ERROR Deployer not found: git

此时，就要检查为什么出现这种错误了，解决方法是上网百度寻求解决方案，还有一种就是靠自己的经验了。我发现在我修改文件_config.yml的时候发现

# Deployment
## Docs: http://hexo.io/docs/deployment.html
deploy:
      type:

发现有个word文档说明，而且我在执行hexo deploy出错，说明解决方法肯定在word链接中，结果是答案真的就在这里面。在浏览器中输入 http://hexo.io/docs/deployment.html
然后按照网页上的文档说明，继续在终端上执行命令

npm install hexo-deployer-git --save

最后在终端上执行命令

hexo deploy

OK，成功了！结果如下

注意：每次修改本地文件后，都要在hexo博客所在的目录下，执行命令 hexo generate 保存；想要上传文件到Github时，就应该在hexo博客所在的目录下先执行命令 hexo generate 保存之后，再执行命令 hexo deploy

Tips:

常用指令

hexo server
hexo generate
hexo deploy

可以简写成

hexo s
hexo g
hexo d

那么静态化 + 部署就可以写成

hexo d -g

######安装更新插件 rss site-map之类的
在终端下依次执行如下命令:

npm install hexo-generator-index --save
npm install hexo-generator-archive --save
npm install hexo-generator-category --save

npm install hexo-deployer-git --save
npm install hexo-deployer-heroku --save
npm install hexo-deployer-rsync --save
npm install hexo-deployer-openshift --save
npm install hexo-renderer-marked@0.2.5 --save
npm install hexo-renderer-stylus@0.2.3 --save
npm install hexo-generator-feed@1.0.2 --save
npm install hexo-generator-sitemap@1.0.1 --save

######装完之后去全局配置文件 _config.yml 修改参数
index_generator:
per_page: 10 ##首页默认10篇文章标题 如果值为0不分页

archive_generator:
      per_page: 10 ##归档页面默认10篇文章标题
      yearly: true  ##生成年视图
      monthly: true ##生成月视图

tag_generator:
      per_page: 10 ##标签分类页面默认10篇文章

category_generator:
       per_page: 10 ###分类页面默认10篇文章


feed:
      type: atom ##feed类型 atom或者rss2
      path: atom.xml ##feed路径
      limit: 20  ##feed文章最小数量

deploy:
  type: git ##部署类型 其他类型自行google之
  repo: <repository url> ##git仓库地址
  branch: [branch] ##git 页面分支
  message: [message] ##git message建议默认字段update 可以自定义
Hexo主题设置

一个简洁优雅的hexo主题 : hexo-theme-yilia。 安装

git clone https://github.com/litten/hexo-theme-yilia.git themes/yilia

配置：修改hexo根目录下的 _config.yml ：

theme: yilia

更新

cd themes/yilia
git pull

######网上有很多hexo的主题,请根据个人的爱好风格来选择适合自己的一款主题.(可以尝试下next主题: git clone https://github.com/iissnan/hexo-theme-next themes/next)

6. 开始写自己的第一篇blog

######First Blog
cd ~/Desktop/DoubleFun_HexoBlog
hexo new firstblog
~/Desktop/DoubleFun_HexoBlog/source/_posts 下生成 firstblog.md 文件
编辑 firstblog.md

title: 在GitHub上搭建基于Hexo的静态Blog   # 文章题目
date: 2015-09-28 16:30:44                 #文章生成时间
categories:                                #文章分类目录 可以省略
    - Web
tags:                                    # 标签(多个标签如下所示)
    - Hexo
    - GitHub
    - Disqus
description:                             #你对本页的描述 可以省略
 ---
 这里开始使用markdown格式输入你的正文

多标签注意语法格式 如下:

tags:
    - 标签1
    - 标签2
    - 标签3
    - etc...

想在首页文章预览添加图片可以添加photo参数 这个fancybox可以省略 如下:

photos:
      - http://xxx.com/photo.jpg

######正文中可以使用< !–more–>设置文章摘要 如下:

以上是文章摘要
<!--more-->
以下是余下全文

more以上内容即是文章摘要，在主页显示，more以下内容点击『> Read More』链接打开全文才显示。。
